В репозитории существует каталог .git, содержащий внутреннюю информацию.
Иногда, при работе с СКВ, возникает ошибка:
  fatal: Unable to create '.git/index.lock': File exists.
index.lock создаётся во время операций с git. Например, при вводе git add *
такой файл создаётся, и существует до тех пор, пока команда не будет
выполнена до конца. Это гарантирует, что в репозитории не произойдёт
одновременных конфликтующих между собой изменений.
Эта ошибка дословно значит, что при попытке работы с git, какой-то процесс
уже работает в этом репозитории, и параллельно создать другой невозможно.
Для решения нужно просто удалить .git/index.lock.
