Сокеты Беркли - API для межпроцессорного взаимодействия.
Эта технология может работать со множеством устройств ввода-вывода,
драйверов. Доступ к интерфейсу сокетов осуществляется на трёх
уровнях, самым мощным является уровень "сырых сокетов".

Хедеры:
    <sys/socket.h> - Базовые функции сокетов и структуры данных.
    
    <netinet/in.h> - Семейство адресов и протоколов PF_INET и
                     PF_INET6. Широко используется, включает в
                     себя IP-адреса и номера портов TCP и UDP.

    <sys/un.h>     - Семейство адресов PF_UNIX и PF_LOCAL.
                     Используется для межпроцессорного взаимодей-
                     ствия. В компьютерных сетях не применяется.

    <arpa/inet.h>  - Функции для работы с числовыми IP-адресами.

   	<netdb.h>      - Функции для преобразования протокольных имён
   	                 в имя хостов и числовые адреса.

Структуры:
    sockaddr - структура адреса.
    Её подвиды:
      sockaddr_in
      sockaddr_in6
      in_addr
      in6_addr

Функции:
    int socket(int domain, int type, int protocol)
      Создаёт конечную точку соединения и возвращает дескриптор.
      Аргументы:
        domain   - Семейство протоколов, например PF_INET,
                   PF_INET6, PF_UNIX для локальных сокетов (файл)

        type     - SOCK_STREAM    - TCP
                   SOCK_DGRAM     - UDP
                   SOCK_SEQPACKET - Служба последовательных пакетов
                   SOCK_RAW       - "Сырой" сокет

        protocol - Транспортный протокол (IPPROTO_(TCP|STCP|UDP)).

      Возвращаемое значение:
        -1 в случае ошибки, иначе файловый дескриптор сокета.

    struct hostnet* getbyhostname(const char* name);
    struct hostnet* getbyhostaddr(const void* addr,
                                         int  len,
                                         int  type);

      Возвращается структура, описывающая интернет-узел по имени
      или адресу. Эта функция просматривает /etc/hosts.
      Аргументы:
        name    - Имя хоста, например "www.google.com"
        addr    - Указатель на struct in_addr
        len     - Длина addr в байтах
        type    - Тип адресов хоста, например PF_INET

      Возвращаемое значение:
        Корректная структура, иначе NULL. Можно также проверить
        h_errno чтобы получить описание ошибки.

    int connect(                  int  sockfd,
                const struct sockaddr* serv_addr,
                            socklen_t  addrlen);

      Устанавливает соединение с сервером. 

      Возвращаемое значение:
        0 в случае успеха, иначе -1.

    int bind(                  int  sockfd,
             const struct sockaddr* my_addr,
                         socklen_t  addrlen);

      Связывает сокет с конкретным адресом. Когда сокет создаётся
      с помощью socket(), он ассоциируется с семейством адресов,
      но не с конкретным адресом. До того, как сокет сможет
      принять входящие соединения, он должен быть связан с
      адресом.

      Аргументы:
        sockfd     - Дескриптор, представлящий сокет
        
        serv_addr  - Указатель представляющий адрес, к которому
                     осуществляется привязка.

        addrlen    - Длина структуры serv_addr.

      Возвращаемое значение:
        0 при успехе, иначе -1.

    int listen(int sockfd, int backlog);

      Подготавливает сокет к принятию входящих сообщений.
      Применимо только к типам SOCK_STREAM и SOCK_SEQPACKET.

      Аргументы:
        sockfd    - Дескриптор сокета.
        backlog   - Число соединений, способных быть обработанными
                    в любой момент времени.   

      Возвращаемое значение:
        0 в случае успеха, иначе -1.

    int accept(            int  sockfd,
               struct sockaddr* cliaddr,
                     socklen_t* addrlen);

      Принятие запроса на установление соединения.

      Аргументы:
        sockfd    - Дескриптор слушающего сокета.
        
        cliaddr   - Указатель на структуру sockaddr для принятия
                    информации об адресе клиента.

        addrlen   - Размер структуры.

      Возвращаемое значение:
        Дескриптор сокета, связанного с соединением, иначе -1.
