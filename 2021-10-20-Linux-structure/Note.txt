Ядро Linux - своего рода виртуальная машина для процессов. Процессу нужен
абстрагированный от уровня оборудования виртуальный интерфейс. Кроме того
ядро реализует многозадачность, задачей которого является правильное расп-
ределение ресурсов между всеми запущенными процессами.

Основные пять компонентов ядра:
  1) Process Scheduler - отвечает за контроль над доступом процессов к CPU.
     Задача этого модуля сводится к правильному распределению нагрузки
     между всеми запущенными процессами.

  2) Memory Manager - отвечает за безопасный доступ множества процессов к
     физической памяти компьютера. Кроме того, обеспечивает работу вирту-
     альной памяти, что позволяет процессам использовать больше памяти, чем
     доступно в системе. Выделенная, но не используемая память вытесняется
     за пределы файловой системы и при необходимости - возвращаются обратно
     в память.

  3) Virtual File System - предоставляет абстрагированный файловый интерфейс
     для всех устройств. VFS поддерживает разные файловые системы для сов-
     местимости с другими ОС.

  4) Network Interface - обеспечивает работу с сетевым оборудованиям и
     протоколами.

  5) Inter-Process Communication - поддержка process-to-process механизмов.

Центральное звено - планировщик процессов. Все остальные модули останавлива-
ют и возобновляют процессы. Например сетевой интерфейс может приостановить
выполнение для получения сетевого пакета, после этого он передаёт управление
обратно планировщику и работа возобновляется.

Однако есть и другие не менее важные зависимости:
  Scheduler <-> MMU - для согласования памяти между процессами.
  IPC <-> MMU       - для поддерживания механизма общей памяти.
  VFS <-> Net       - для обеспечения работы VFS.
  MMU <-> VFS       - для поддержки файлов подкачки (swap).
